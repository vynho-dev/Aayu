# Aayu App UI Kit

Click-through recreation of the MVP hero flow from `Design_System.md` §4–5 and PRD §11.1/11.3: Landing → Auth → Patient selector → Consent → Upload → Processing → Claim result → (Policy Q&A | Vault).

No screenshots, Figma, or codebase existed for this repo — every screen is built directly from the written spec's IA, component list, and copy rules (Design_System.md §4, §6, §7). Scheme match is shown as a card inside Vault, per the spec's "can be a card on the Vault or dashboard" allowance.

Open `index.html` for the interactive phone-frame demo.
