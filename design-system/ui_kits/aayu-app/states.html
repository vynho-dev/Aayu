UX Blueprint · Screen states
Every screen, every situation
Developers need to know what to show when there's no data, while work is happening, when something fails, and when it succeeds — not just the happy path.
Claim / Upload

The hero task — all four states

The most important flow. Errors are specific and never dead-end; every failure offers a way forward.

Empty
First upload — an invitation with the trust promise up front.
Loading
Names the step ("Finding the clause…"), reassures, shows encryption.
Error
Blurry photo → retake; wrong file → re-upload. No "Error:" jargon.
Success
The result IS the success — calm confirmation + the amount.
Health

Health record states

Empty
"Appa's health record starts here." — an invitation, not "nothing yet."
Loading
Skeleton rows, not a lone spinner.
Populated
Auto-built from uploads: conditions, meds, allergies, timeline.
Schemes

Scheme match states

Loading
Checking a small curated list against the profile.
No match
Explicit, honest empty state — never a blank screen.
Matched
One high-confidence match + "still checking." 3–5 max.
